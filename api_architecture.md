## Lead Ingestion Flow (via n8n)

All website lead submissions are routed through **n8n** before reaching the Portal:

```
Website OrderBuilder.tsx
  ‚Üí POST JSON to n8n webhook (production URL, auto-fallback to test URL)
      ‚Üí n8n workflow processes/validates data
      ‚Üí n8n POST JSON to Portal /api/leads/create (with x-api-key)
          ‚Üí Portal inserts lead into database
```

## Public Endpoints

### `POST /api/leads` ‚Äî Create Lead (**DEPRECATED**)

> ‚öÝÔ∏è **Deprecated.** The website now submits leads through the n8n webhook, which calls `/api/leads/create`. This endpoint can be removed once the n8n flow is fully verified.

**Auth:** None (fully public, CORS-open)  
**Content-Type:** `multipart/form-data`  
**Supabase Client:** Admin (Service Role Key)

---

### `POST /api/leads/create` ‚Äî Create Lead (n8n ‚Üí Portal)

**Auth:** `x-api-key` header (validated against `LEADS_API_KEY` env var)  
**Content-Type:** `application/json`  
**Supabase Client:** Server Client

| Field | Type | Required | Description |
|---|---|---|---|
| `name` | string | Yes | Contact name |
| `email` | string | Yes | Contact email |
| `company` | string | No | Company name |
| `phone` | string | No | Contact phone |
| `summary` | string | No | Request summary (can be generated by n8n) |
| `source` | string | No | Lead source identifier (default: `'api'`) |
| `details` | object | No | Additional order details |
| `file_paths` | string[] | No | Array of Supabase Storage public URLs for uploaded files |

**Response:** `{ success: true, lead: {...} }`

> **n8n mapping:** In the n8n workflow, map `meta.fileUrls` from the webhook payload ‚Üí `file_paths` in the body of the `/api/leads/create` request.

## Admin Endpoints

All admin endpoints validate the calling user has `role === 'admin'` on the server side.

### `POST /api/admin/convert-lead`
Converts a qualified lead into a full client organization:
- Creates new organization with generated access code
- Links lead data to the new organization
- Updates lead status to `converted`

### `POST /api/admin/delete-user`
Removes a portal user:
- Deletes user from `auth.users` via admin client
- Cascades to profile and membership removal

### `POST /api/admin/invite-user`
Invites a user to join an organization:
- Creates auth user or sends invite email
- Optionally pre-links to an organization

## Server Actions (Non-REST)

These use Next.js Server Actions (`'use server'`) called from client components:

| Action | File | Purpose |
|---|---|---|
| `setActiveOrganization` | `app/actions.ts` | Set org cookie + redirect |
| `login` | `app/login/actions.ts` | Email/password authentication |
| `signup` | `app/signup/actions.ts` | New account registration |
| `joinOrganization` | `app/join/actions.ts` | Link to org via access code |
| `forgotPassword` | `app/forgot-password/actions.ts` | Send reset email |
| `resetPassword` | `app/reset-password/actions.ts` | Update password |

## Error Handling

- API routes return standard JSON error responses: `{ error: "message" }`
- HTTP status codes: `400` (bad request), `500` (server error)
- Webhook failures are logged but do not fail the lead creation request
